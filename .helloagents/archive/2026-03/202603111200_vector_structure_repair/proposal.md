# 方案: vector_structure_repair

## 背景

第二轮已经显著恢复填充颜色和透明度，但用户最新截图表明主样本在结构识别上仍明显失真。

用户已确认第三轮目标：

- 结构正确性优先
- 一步一步推进
- 优先修复三块重点区域：
  - 左侧 `Polygenic` 扇形连线区
  - 中部 `Stratagenic` 三组彩色团块
  - 右侧 `Omnigenic` 大轮廓与内部层级

## 根因诊断

1. 左侧区域目前主要是一个大 `stroke` 线束加若干 Hough 圆点，没有形成稳定的“节点层 + 连线层 + 汇聚中心”表达。
2. 中部三组彩色团块虽然已经恢复颜色，但内部孔洞、装饰碎片和节点没有分层，视觉上接近碎片拼贴。
3. 右侧大轮廓 `region-000` 已存在，但内部放射状切片、孔洞层和节点层仍混杂在同一层级。
4. 当前 `detect_structure` 只理解 `box / arrow / container`，不足以表达这类信息图中的 `fan / cluster / hull` 结构。

## 方案选择

已选定方案 A：分层轮廓重建。

### 核心思路

针对重点区域，把当前“单层 contour 重建”升级为“外轮廓层 + 内部孔洞层 + 节点/装饰层 + 连接线层”的分层重建。

### 实施方向

1. 先补失败测试，锁定主样本三个重点区域的结构特征。
2. 在 `scene_graph / vectorize_region` 中引入更强的 region 分层表达。
3. 视需要在 `segment / detect_structure` 中增加更稳的 fan / cluster / hull 辅助判断。
4. 以主样本优先验证，同时确保 `F2.png`、`orr_signature.png` 等现有样本不明显退化。

## 验收

- 左侧区域不再退化为单个大线束 + 零散圆点
- 中部三组团块保留大色块轮廓，并能区分内部空洞/装饰
- 右侧区域保留大外轮廓，并减少内部碎裂感
- `pytest -q` 继续全绿
- `./picture` 回归样本不明显退化

## 实际结果

- 已补第三轮失败测试并完成红转绿
- 已为复杂带孔填充区域引入 `evenodd path`
- 已新增 `fan` 结构检测与导出重建
- 最新验证结果为 `101 passed, 1 warning`
